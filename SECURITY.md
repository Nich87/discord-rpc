# Security Policy / セキュリティポリシー

## Supported Versions / サポートバージョン

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability / 脆弱性の報告

**English:**

If you discover a security vulnerability in this project, please report it responsibly. **Do not open a public GitHub issue.**

Instead, please report it via one of the following methods:

1. **GitHub Security Advisories**: Use the [Security Advisories](https://github.com/Nich87/discord-rpc/security/advisories/new) feature to privately report the vulnerability.
2. **Email**: Contact the maintainer directly (see the profile of [@Nich87](https://github.com/Nich87) for contact information).

### What to include / 含めるべき情報

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes (if applicable)

### Response Timeline / 対応タイムライン

- **Acknowledgment**: We will acknowledge receipt of your report within **48 hours**.
- **Assessment**: We will assess the vulnerability and provide an initial response within **7 days**.
- **Fix & Disclosure**: Once confirmed, we aim to release a fix and disclose the vulnerability responsibly within **30 days**.

---

**日本語:**

このプロジェクトにセキュリティ脆弱性を発見した場合は、責任を持って報告してください。**公開のGitHub Issueを作成しないでください。**

代わりに、以下のいずれかの方法で報告してください：

1. **GitHub Security Advisories**: [Security Advisories](https://github.com/Nich87/discord-rpc/security/advisories/new)機能を使用して、非公開で脆弱性を報告してください。
2. **メール**: メンテナーに直接連絡してください（[@Nich87](https://github.com/Nich87)のプロフィールから連絡先をご確認ください）。

### 含めるべき情報

- 脆弱性の説明
- 問題を再現する手順
- 脆弱性の潜在的な影響
- 修正案（該当する場合）

### 対応タイムライン

- **受領確認**: 報告の受領を**48時間以内**に確認します。
- **評価**: 脆弱性を評価し、**7日以内**に初期対応を行います。
- **修正と開示**: 確認後、**30日以内**に修正をリリースし、責任ある開示を行うことを目指します。
